# Kanwas Frontend DOX

## Purpose

Vite + React app for the Kanwas workspace experience: auth routes, workspace routing, multiplayer canvas, chat/timeline, BlockNote-based notes, public shares, embed bootstrap, skills UI, and organization settings.

## Ownership

- `src/App.tsx` owns browser routing, providers, public/share/embed/CLI authorization pages, and protected workspace routes.
- `src/pages/WorkspacePage.tsx` composes the live workspace surface, sidebar, canvas, chat, search, editor manager, keyboard flow, and route-to-canvas state.
- `src/providers/` owns app-level state providers for auth, keyboard, workspace Yjs connection, chat, selected nodes, project state, public notes, and theme.
- `src/components/canvas/` owns React Flow canvas rendering, canvas mutations, node projection, grouping, sections, viewport, selection, paste/import interactions, and node UIs.
- `src/components/note-editors/`, `src/hooks/useNoteBlockNoteBinding.ts`, and `src/lib/blocknote-*` own BlockNote editor binding and collaboration behavior.
- `src/api/` owns frontend API wrappers around Tuyau/backend routes and Socket.IO event room helpers.
- `src/store/` owns local UI and realtime markdown write state.

## Local Contracts

- `src/api/client.ts` imports `backend/api` generated by Tuyau. Backend route/controller/validator changes require `pnpm --filter backend codegen` before frontend types are reliable.
- `src/providers/auth/AuthProvider.tsx` owns bearer-token persistence and optional local-dev default login. Keep default auto-login opt-in via `VITE_DEFAULT_AUTH_ENABLED=true` (paired with backend `DEFAULT_USER_LOGIN_ENABLED=true`) so normal and multi-user auth flows can remain unauthenticated by default.
- `WorkspaceProvider` creates the canonical frontend `Y.Doc`, `valtio-y` proxy over the `state` map, `WorkspaceSocketProvider`, and `WorkspaceContentStore`. Do not duplicate workspace connection state elsewhere.
- `WorkspaceSocketProvider` comes from `shared/workspace-provider`; the socket token is fetched through `useYjsSocketToken`. Token and protocol changes must be coordinated with backend and yjs-server.
- BlockNote notes are attached Yjs subdocs from `shared/note-doc`. Preserve fragment identity, awareness acquisition/release, and undo manager behavior when editing note binding code.
- `frontend/src/lib/ydoc-utils.ts` warns that converting blocks into a Y.XmlFragment is for first import only; do not use it to rehydrate a collaborated document after history exists.
- Tailwind is v4 via `@tailwindcss/vite` and `src/index.css`; follow the root rule: no `tailwind.config.ts` and no `@apply`.
- Vite dev server is configured for `0.0.0.0:5173`; Playwright uses `http://localhost:5173` as `baseURL`.

## Work Guidance

- Dev: `pnpm --filter frontend dev`
- Build: `pnpm --filter frontend build`
- Typecheck used by package script: `pnpm --filter frontend typecheck`
- Full frontend typecheck: `pnpm --filter frontend typecheck:all`
- Unit/component tests: `pnpm --filter frontend test`
- Watch tests: `pnpm --filter frontend test:watch`
- E2E tests: `pnpm --filter frontend test:e2e`
- Smoke E2E: `pnpm --filter frontend test:smoke`
- Workspace E2E: `pnpm --filter frontend test:workspace`
- Lint: `pnpm --filter frontend lint`
- Preview built app: `pnpm --filter frontend preview`

## Verification

- Static checks: `pnpm --filter frontend lint` and `pnpm --filter frontend typecheck`
- Unit/component coverage: `pnpm --filter frontend test`
- Browser flows: `pnpm --filter frontend test:smoke`, `pnpm --filter frontend test:workspace`, or `pnpm --filter frontend test:e2e`
- For canvas/editor/collaboration changes, verify in a real browser against the running stack, not only with jsdom tests.
